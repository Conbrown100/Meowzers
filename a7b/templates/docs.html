API Documentation

Route Params Returns
GET /api/posts/ profile_id : integer (optional) - a specific user's profile ID; if omitted, only retrieves the currently logged-in user's posts A JSON array of Post objects authored by the chosen Profile. If the profile_id is invalid this returns [].
GET /api/posts/<int:post_id>/ post_id : integer (required) - a specific post ID A single JSON Post object. If the post_id is invalid, this returns a 404 error.
POST /api/posts/ content : string (required) The newly created JSON Post object
POST /api/posts/<int:post_id>/like/ post_id : integer (required) The liked JSON Post object; if the post does not exist and cannot be liked, this will return a 404.
POST /api/posts/<int:post_id>/unlike/ post_id : integer (required) The unliked JSON Post object; if the post does not exist and cannot be liked, this will return a 404.
GET /api/posts/<int:post_id>/likes/ post_id : integer (required) A JSON array of Profile objects (those who have liked the given post). If the post does not exist and cannot be liked, this will return a 404.
